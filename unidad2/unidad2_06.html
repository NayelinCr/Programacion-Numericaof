1. Ejercicio 1: An√°lisis de Crecimiento de Usuarios

Una startup de tecnolog√≠a registra el n√∫mero acumulado de usuarios activos mensuales (en miles).

Planteamiento y Soluci√≥n Matem√°tica

Datos: f(1)=10, f(2)=15, f(3)=23, f(4)=34, f(5)=48, f(6)=65, f(7)=85. Paso h=1 mes.

C√°lculos realizados:

  • Tasa en Mes 4 (Diferencia Centrada): 12.5 mil usuarios/mes.
  • Tasa en Mes 1 (Adelante): 5 mil usuarios/mes.
  • Tasa en Mes 7 (Atr√°s): 20 mil usuarios/mes.
  • Segunda derivada (Aceleraci√≥n): Constante en 3 mil usuarios/mes¬≤.

Interpretaci√≥n: La startup est√° creciendo de forma acelerada (c√≥ncava hacia arriba).

C√≥digo en R

# EJERCICIO 1: ANALISIS DE CRECIMIENTO DE USUARIOS
# Datos: usuarios mensuales acumulados (en miles)

# --- Datos ---
usuarios <- c(10, 15, 23, 34, 48, 65, 85) # f1...f7
h <- 1 # paso en meses

# 1) Derivadas primeras
# Mes 1: diferencia hacia adelante
fwd_mes1 <- (usuarios[2] - usuarios[1]) / h

# Mes 4: diferencia centrada
central_mes4 <- (usuarios[5] - usuarios[3]) / (2 * h)

# Mes 7: diferencia hacia atras
bwd_mes7 <- (usuarios[7] - usuarios[6]) / h

# 2) Segunda derivada para cada punto (3-puntos)
n <- length(usuarios)
segunda <- numeric(n)

# Extremos y puntos internos
segunda[1] <- (usuarios[3] - 2*usuarios[2] + usuarios[1]) / h^2
for (i in 2:(n-1)) {
  segunda[i] <- (usuarios[i+1] - 2*usuarios[i] + usuarios[i-1]) / h^2
}
segunda[n] <- (usuarios[n] - 2*usuarios[n-1] + usuarios[n-2]) / h^2

# Mostrar resultados
cat("EJERCICIO 1: Diferenciacion Numerica (R)\n")
cat(sprintf("Derivada (mes 1, adelante): %.4g miles usuarios/mes\n", fwd_mes1))
cat(sprintf("Derivada (mes 4, centrada): %.4g miles usuarios/mes\n", central_mes4))
cat(sprintf("Derivada (mes 7, atras): %.4g miles usuarios/mes\n", bwd_mes7))

cat("Segunda derivada (aceleracion) por mes:\n")
for (i in 1:n) {
  cat(sprintf(" Mes %d: %g\n", i, segunda[i]))
}

2. Ejercicio 2: Optimizaci√≥n de Funci√≥n de P√©rdida

Durante el entrenamiento de un modelo de Machine Learning, se registra la funci√≥n de p√©rdida (loss) en cada √©poca.

Planteamiento

Datos: Loss en √©pocas 0, 10, 20, 30, 40, 50. Paso h=10.

  • f'(20) usando diferencia centrada.
  • f''(30) para analizar convergencia.
  • Criterio de parada: |f'| < 0.01.

Soluci√≥n y C√≥digo en R

# EJERCICIO 2: OPTIMIZACION DE LOSS
epocas <- c(0, 10, 20, 30, 40, 50)
loss <- c(2.45, 1.82, 1.35, 1.08, 0.95, 0.89)
h <- 10 # paso

# 1) Derivada centrada en epoca 20 (indice 3)
f_prime_20 <- (loss[4] - loss[2]) / (2 * h)

# 2) Segunda derivada en epoca 30 (indice 4)
f_second_30 <- (loss[5] - 2*loss[4] + loss[3]) / (h^2)

# 3) Calcular derivadas en todas las epocas
n <- length(epocas)
f_prime <- numeric(n)
f_prime[1] <- (loss[2] - loss[1]) / h # Forward
for (i in 2:(n-1)) {
  f_prime[i] <- (loss[i+1] - loss[i-1]) / (2 * h) # Centrada
}
f_prime[n] <- (loss[n] - loss[n-1]) / h # Backward

# Criterio de parada
umbral <- 0.01
cumple <- which(abs(f_prime) < umbral)

# 4) Estimacion lineal f(25)
f_est_25 <- loss[3] + (25 - 20) * f_prime[3]

cat("EJERCICIO 2 Output:\n")
cat(sprintf("f'(20) = %.6g\n", f_prime_20))
cat(sprintf("f''(30) = %.6g\n", f_second_30))
cat(sprintf("Estimacion f(25) approx %.6g\n", f_est_25))

3. Ejercicio 3: An√°lisis de Series Temporales de Ventas

An√°lisis de ventas diarias durante una semana de campa√±a.

Planteamiento

Se busca identificar el d√≠a de mayor aceleraci√≥n, la magnitud de la ca√≠da del jueves y estimar las ventas del lunes siguiente.

C√≥digo en R

# EJERCICIO 3: ANALISIS DE VENTAS DIARIAS
dias <- c("Lun", "Mar", "Mie", "Jue", "Vie", "Sab", "Dom")
ventas <- c(45, 52, 61, 58, 73, 89, 95)
h <- 1

n <- length(ventas)
velocidad <- numeric(n)
velocidad[1] <- (ventas[2] - ventas[1]) / h
for (i in 2:(n-1)) { velocidad[i] <- (ventas[i+1] - ventas[i-1]) / (2*h) }
velocidad[n] <- (ventas[n] - ventas[n-1]) / h

# Segunda derivada (aceleracion)
aceleracion <- rep(NA, n)
for (i in 2:(n-1)) {
  aceleracion[i] <- (ventas[i+1] - 2*ventas[i] + ventas[i-1])
}

# Dia con mayor aceleracion
max_acc_index <- which(aceleracion == max(aceleracion, na.rm=TRUE))
dia_max_acc <- dias[max_acc_index]

# Extrapolacion lunes siguiente
ventas_lunes_sig <- ventas[n] + velocidad[n] * 1

cat("EJERCICIO 3 Output:\n")
cat(sprintf("Dia con mayor aceleracion: %s (%g $k/dia^2)\n", dia_max_acc, max(aceleracion, na.rm=TRUE)))
cat(sprintf("Estimacion ventas lunes siguiente: %g $k\n", ventas_lunes_sig))

# Funcion auxiliar diferencia centrada
centered_diff <- function(x, map, h=1) {
  x_plus <- as.character(x + h)
  x_minus <- as.character(x - h)
  return((map[x_plus] - map[x_minus]) / (2 * h))
}

# Calculo en x=0, x=-2, x=2

deriv_neg2 <- centered_diff(-2, sigma_vals)
deriv_pos2 <- centered_diff(2, sigma_vals)

6. Ejercicio 6: Tasa de Conversi√≥n (ROI)

An√°lisis de rendimientos decrecientes en inversi√≥n publicitaria.

Soluci√≥n

Se calcula el ROI marginal. Se observa que f''(15) es negativo (-0.012), indicando rendimientos decrecientes. La recomendaci√≥n es no aumentar el gasto m√°s all√° de $25k.

C√≥digo en R

# EJERCICIO 6: ROI MARGINAL
gasto_k <- c(0, 5, 10, 15, 20, 25)
conversion <- c(2.1, 3.8, 5.2, 6.1, 6.7, 7.0)
h <- 5
n <- length(gasto_k)

roi <- numeric(n)
roi[1] <- (conversion[2] - conversion[1]) / h
for(i in 2:(n-1)) roi[i] <- (conversion[i+1] - conversion[i-1]) / (2*h)
roi[n] <- (conversion[n] - conversion[n-1]) / h

# Segunda derivada en 15k (indice 4)
f2_15 <- (conversion[5] - 2*conversion[4] + conversion[3]) / (h^2)

cat("EJERCICIO 6 Output:\n")
cat(sprintf("ROI marginal en $25k: %.4g %%/$k\n", roi[n]))
cat(sprintf("f''(15) = %.6g (Rendimientos decrecientes)\n", f2_15))

7. Ejercicio 7: Feature Engineering con Derivadas

Creaci√≥n de nuevas caracter√≠sticas (velocidad y aceleraci√≥n) para un sensor de temperatura industrial.

C√≥digo en R

n <- length(temp)

# Velocidad (1ra derivada)

# Alertas
umbral <- 0.8
alertas <- which(d1 > umbral)

cat("EJERCICIO 7 Output:\n")
print(data.frame(Tiempo=0:7, Temp=temp, Velocidad=round(d1,3), Aceleracion=round(d2,3)))
